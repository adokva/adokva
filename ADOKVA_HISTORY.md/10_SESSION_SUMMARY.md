# Назначение

Данный документ хранит краткую информацию о последней рабочей сессии проекта ADOKVA.

Он используется для быстрого восстановления контекста после перерыва и позволяет за несколько минут понять, где была остановлена разработка.

# Рабочий стиль проекта

Во время разработки используется следующий формат взаимодействия.

Пользователь пишет команду:

```
йй
```

Это означает:

- режим активной разработки;
- минимум обсуждений;
- максимум готовых решений;
- полный код файлов;
- пошаговые инструкции по вставке.

Каждый ответ содержит:

- дату;
- время;
- название файла;
- действия пользователя;
- готовый код.
