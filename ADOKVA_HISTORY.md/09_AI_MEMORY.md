# AI MEMORY

Версия документа: 1.0

---

# Назначение

Этот документ предназначен для быстрого погружения нового чата ChatGPT в проект ADOKVA.

Его задача — максимально сократить потерю контекста между сессиями и обеспечить последовательное продолжение разработки.

Этот документ не заменяет архитектурную документацию, а служит краткой "оперативной памятью" проекта.

---

# Что такое ADOKVA

ADOKVA — долгосрочный проект интерактивной 3D-платформы, объединяющей Землю, Солнечную систему и космос в единой сцене.

Главная цель проекта — создать масштабируемую архитектуру, которую можно развивать много лет без необходимости переписывать фундамент.

Проект создаётся как полноценное приложение, а не как демонстрационный пример.

# Текущая архитектура

Каждый крупный объект располагается в собственном компоненте.

Например:

- Earth
- Sun
- Moon
- Mercury
- Mars

Каждая система имеет собственную ответственность.

Данные постепенно централизуются.

Логика отделяется от интерфейса.

# Как продолжать разработку

Перед началом работы необходимо:

1. Прочитать `00_START_HERE.md`.
2. Прочитать `02_CURRENT_STATE.md`.
3. Проверить `07_NEXT_TASK.md`.
4. Ознакомиться с последними изменениями в `CHANGELOG.md`.
5. Только после этого приступать к следующей задаче.

---

# Формат совместной работы

Во время разработки используется следующий стиль:

- пользователь пишет команду **«йй»** — это означает режим активной разработки;
- ChatGPT отвечает готовыми файлами целиком;
- перед каждым ответом указываются дата и время;
- обязательно пишется, в какой файл вставлять код;
- обязательно указывается последовательность действий.

---

# Правила генерации кода

Предпочтительно:

- полный файл вместо отдельных фрагментов;
- сохранение существующей архитектуры;
- отсутствие необоснованных переписываний;
- минимальные изменения при исправлении ошибок;
- подробные инструкции по вставке.

---

# Что нельзя делать

Не следует:

- предлагать переписывать уже работающие системы без причины;
- смешивать данные, интерфейс и вычисления;
- нарушать структуру каталогов;
- добавлять временные решения, которые ухудшают архитектуру.

---

# Если необходимо продолжить проект в новом чате

Порядок действий:

1. Изучить этот документ.
2. Изучить `CURRENT_STATE.md`.
3. Изучить `NEXT_TASK.md`.
4. Проверить последние изменения.
5. Продолжить выполнение первой незавершённой задачи.
