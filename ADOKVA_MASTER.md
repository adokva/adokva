# 1. ГЛАВНАЯ ИДЕЯ ADOKVA

ADOKVA — интерактивная социальная 3D-платформа вокруг Земли, людей, городов и других миров.

Главная идея:

Человек заходит в ADOKVA и видит живую интерактивную Землю.

Он может указать:

- где родился;
- где живёт сейчас;
- где был;
- левша он или правша;
- какие страны и города посещал;
- в будущем — на каких планетах или мирах он «был».

После этого человек сможет увидеть:

- сколько людей родились там же, где он;
- сколько его земляков живут рядом с ним сейчас;
- сколько его земляков живут в других городах;
- сколько его земляков живут в других странах;
- куда люди из его родного города разъехались по миру;
- откуда приехали люди, которые сейчас живут рядом с ним;
- в будущем — распределение людей по другим планетам и мирам.

Главная социальная цепочка:

Земля
→ место рождения
→ место проживания
→ земляки
→ люди по всему миру
→ путешествия
→ другие миры
→ планеты
→ AI

Ключевая идея:

«Я родился здесь.
Сейчас я живу здесь.
Мои земляки разлетелись по всему миру.
ADOKVA позволяет увидеть их на одной живой Земле.»

Земля должна стать не просто красивым 3D-глобусом.

Земля — это визуальный интерфейс человеческих связей.

# 2. РЕГИСТРАЦИЯ И СООБЩЕНИЯ

На текущем этапе принято решение:

НЕ создавать сложную обязательную регистрацию.

НЕ создавать личные сообщения и внутренний мессенджер.

Причина:

- не усложнять ранний прототип;
- не создавать преждевременно сложную систему аккаунтов;
- не создавать лишние вопросы хранения пользовательских данных;
- не усложнять модерацию;
- не усложнять юридическую сторону проекта в разных странах.

На первом этапе пользователь должен иметь возможность максимально просто зайти в ADOKVA и пользоваться основной идеей платформы.

Возможная будущая система:

- минимальный профиль;
- необязательный аккаунт;
- сохранение данных только при необходимости;
- более сложную регистрацию рассматривать позже.

Юридические и privacy-требования конкретных стран должны быть отдельно изучены перед публичным запуском и реальным сбором пользовательских данных.

# 4. СИСТЕМА ЗЕМЛЯКОВ

Одна из главных будущих функций ADOKVA.

Пользователь указывает место рождения.

ADOKVA показывает:

- сколько людей родились в том же городе;
- сколько людей родились в той же стране;
- где эти люди живут сейчас;
- сколько земляков живут в городе пользователя;
- сколько земляков живут в других странах.

Пример:

Пользователь родился в Москве.

Сейчас живёт в Дубае.

ADOKVA сможет показать:

Москва
→ сколько зарегистрированных людей родились в Москве

Москва → Дубай
→ сколько москвичей сейчас живут в Дубае

Москва → США
→ сколько москвичей живут в США

Москва → Германия
→ сколько москвичей живут в Германии

В будущем это может визуализироваться непосредственно на глобусе:

- точки;
- маркеры;
- линии;
- дуги;
- потоки людей между городами.

# 16. НАВИГАЦИЯ МЕЖДУ МИРАМИ

Главное правило навигации:

тап на объект
→ объект выходит на передний план.

Повторный тап на активный объект
→ объект возвращается
→ Земля выходит в центр.

Эта система уже используется для:

- Луны;
- Солнца.

Она должна стать единой системой для всех будущих миров.

Не использовать отдельную кнопку:

«Вернуться к Земле»

если возврат можно выполнить повторным тапом.

создать надёжный Camera Engine версии 2.

Особое внимание:

- горизонт не должен заваливаться;
- север должен оставаться сверху;
- после полёта OrbitControls не должны наследовать крен;
- камера должна двигаться вокруг Земли;
- Земля не должна поворачиваться ради выбранной цели.

Последнее выполненное изменение:

в FocusOrbitControls.tsx добавлена стабилизация:

- camera.up направляется по мировому северу;
- камера повторно смотрит на центр Земли;
- стабилизация выполняется после OrbitControls.

Требуется проверить в браузере:

- держится ли горизонт;
- отсутствует ли крен;
- правильно ли работает ручное вращение после полёта.

# 10. ПРАВИЛО ЙЙ

Пользователь проекта не является программистом.

Поэтому все инструкции должны быть максимально точными.

Разрешён только следующий формат работы:

1. Указать один файл.
2. Написать полный путь Windows.
3. Дать полный готовый код файла.
4. Пользователь нажимает Ctrl+A.
5. Пользователь вставляет код через Ctrl+V.
6. Пользователь сохраняет файл через Ctrl+S.
7. Проверяется панель Problems.
8. Следующий шаг выполняется только после Problems = 0.

Запрещённые инструкции:

- «удали несколько строк»;
- «вставь между этими функциями»;
- «найди нужное место»;
- «измени только одну часть»;
- «добавь импорт наверху»;
- «перенеси этот блок».

Все изменения выполняются полной заменой одного файла.
